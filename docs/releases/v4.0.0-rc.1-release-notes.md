# v4.0.0-rc.1 Release Notes

## What changed

- replaced the older embedded session-plus-state consumer model with a single app-owned `EmbeddedServer` observable surface
- moved the embedded lifecycle entrypoint onto that public object, so app consumers now create the object, call `liftoff()`, bind SwiftUI directly to its properties, and call its app-facing control actions there as well
- tightened the package surface so transport-local payload models and CLI-only helper types are no longer exported as part of the library story
- refreshed the README and DocC articles so the documented embedding path now matches the intended public API and ownership model

## Breaking changes

- the app-consumer entrypoint is now `EmbeddedServer` instead of the earlier `EmbeddedServerSession` plus `ServerState` split
- embedded lifecycle code should now call `liftoff()` on the consumer-owned object and bind directly to its observable properties instead of retaining one object for lifecycle and another for UI state
- code that referenced previously public internal helper or transport model types from the library product will need to migrate away from those now-hidden symbols

## Migration or upgrade notes

- create and retain one `EmbeddedServer` per app-owned embedded server instance
- drive SwiftUI directly from `EmbeddedServer` properties such as runtime overview, playback state, queues, voice profiles, and related consumer-facing controls
- use `land()` for teardown on the same object that performed `liftoff()`
- if you were only embedding the server and not intentionally reaching into CLI or transport implementation details, the new shape should be simpler even though it is a breaking rename
