# Wrapper Contract: `speak-with-profile`

Adapter contract for profile-first speech generation with built-in `$speech` as default synthesis backend.

## Front-door rule
- Speech tasks should enter through `$speak-with-profile`.
- This skill resolves policy/profiles, then delegates synthesis or prepares a deterministic local CLI invocation.

## Execution modes
- `delegate` (default): built-in `$speech` performs generation.
- `local-cli` (fallback): `scripts/speak_with_profile.py` invokes a compatible speech CLI. The default `--tts-cli-path` value expects `scripts/text_to_speech.py` in the working tree, but callers may pass an explicit path.

## Workflow configuration precedence
1. Explicit user input and wrapper flags.
2. Skill-local workflow customization in `config/customization.yaml`, when present.
3. Skill-local defaults in `config/customization.template.yaml`.
4. Baseline workflow defaults.

Skill-local customization guides agent decisions and explicit wrapper arguments. The current wrapper script auto-loads supported customization values from these files. `preferredExecutionMode` remains documentation-only because the script does not implement `delegate` execution.

## Command signature (local-cli)
```bash
speak-with-profile \
  --text-file <path> | --text "<text>" \
  [--profile <id>] \
  [--profiles-file <path>] \
  [--voice <voice>] \
  [--instructions "<text>"] \
  [--speed <float>] \
  [--format mp3|wav|opus|aac|flac] \
  [--tts-cli-path <path>] \
  [--out <path>]
```

## Delegation payload (to `$speech`)
When delegating, pass resolved fields explicitly:
- `input` (verbatim or adapted by user request)
- `voice`
- `instructions`
- `speed`
- `response_format`
- output target/path intent

## Output contract
- `delegate`: return a user-visible result that includes disclosure text and the resolved speech settings relevant to the request.
- `local-cli`: write the audio file and adjacent manifest, then report both paths plus the playback result when playback is requested.

## M1 precedence
1. Explicit wrapper flags.
2. Selected profile values.
3. `default_profile` values.
4. Baseline defaults.

## Baseline defaults
- `model`: `gpt-4o-mini-tts-2025-12-15`
- `voice`: `cedar`
- `speed`: `1.0`
- `response_format`: `mp3`
- `defaultProfilesFile`: `config/speech/profiles.yaml`
- `localCli.defaultOutputDir`: `output/speech`
- `localCli.defaultPlaybackBackend`: `none`
- `localCli.autoplayGeneratedAudio`: `false`

## Disclosure policy
For any produced speech output, include disclosure text in user-facing context.
- Default: `This audio was generated by AI text-to-speech.`
- If profile `disclosure` is present, prefer that value.

## Input limits
- Keep each request input at or below 4096 characters for local-cli path.
- For longer input, either fail fast with guidance or chunk deterministically.

## Manifest contract
Write `<audio_filename>.manifest.json` with these required fields:
- `timestamp`
- `input_source`
- `profile_id`
- `effective_voice`
- `effective_instructions`
- `effective_speed`
- `summarize_mode`
- `target_chars`
- `playback_backend`
- `playback_result`
- `audio_output_path`
- `text_sha256`

Optional:
- `summary_sha256`
- `errors`
- `disclosure`
- `backend_mode` (`delegate` or `local-cli`)
- `profile_source` (`file` when resolved from a profiles file)

## Failure semantics
- Missing API key: fail non-zero (local-cli) or block delegate call.
- Unknown profile id: fail with known profile list.
- Invalid profiles file: fail with field-level failures.
- Generation backend failure: propagate non-zero/error state.
