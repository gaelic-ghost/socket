# Fix Policies

- Keep edits bounded to the target `CONTRIBUTING.md`.
- Preserve existing section bodies whenever they already satisfy the schema.
- Add missing sections with grounded default guidance, not project-fiction.
- Add setup or verification commands only when repo signals support them directly.
- Prefer neutral contributor guidance when runtime commands cannot be inferred safely.
- Do not synthesize environment variable names, service names, branch naming schemes, or release workflow rules.
