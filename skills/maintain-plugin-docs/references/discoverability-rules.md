# Discoverability Rules

For public curated repos, include:

- `npx skills add <owner/repo>`
- `npx skills add <owner/repo> --all`
- `npx skills add <owner/repo> --skill <skill-name>` for individual install examples
- `## Table of Contents` near the top (compact, H2 links only)
- `## Install individually by Skill or Skill Pack`
- `## Update Skills` section with `npx skills check` and `npx skills update`
- at least 3 realistic `npx skills find` examples
- `## More resources for similar Skills` wrapper section
- `### Find Skills like these with the \`skills\` CLI by Vercel — [vercel-labs/skills](https://github.com/vercel-labs/skills)`
- `### Find Skills like these with the \`Find Skills\` Agent Skill by Vercel — [vercel-labs/agent-skills](https://github.com/vercel-labs/agent-skills)`
- allow legacy/variant spelling for this second heading (for example `Find Skills` Skill by Vercel) and normalize to canonical heading
- the anchor line `Then ask your Agent for help finding a skill for "" or ""` before any optional extra `###` subsections

Include links:

- `https://github.com/vercel-labs/skills`
- `https://github.com/vercel-labs/agent-skills`
- `https://skills.sh/`
