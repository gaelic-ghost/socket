# Review And Validation

Review each glass treatment against its job:

1. Is the surface actionable or meaningfully elevated?
2. Are grouped glass elements actually related and consistently shaped?
3. Does the fallback remain legible and operable on the oldest supported OS?
4. Does text retain sufficient contrast in Light and Dark appearances?
5. Can keyboard, pointer, VoiceOver, Dynamic Type, and reduced-motion users complete the same action?

Validate on a real supported destination when possible. Report visual observations separately from compile/build evidence, and report fallback validation separately from the current-system glass path.
