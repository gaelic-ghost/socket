# Official Web Docs

Use official Apple or Swift web docs when Xcode MCP docs are unavailable, when Dash is unavailable or insufficient, when open source Swift repositories or generated docs do not answer the question, or when the user explicitly prefers official web documentation.

- Prefer official vendor documentation first when a local docs path is unavailable.
- For open source Swift projects, prefer the official GitHub/source repository, generated DocC, release notes, or checked-out source before treating an unreadable Apple Developer web page as the evidence source.
- Do not use generic no-JS web search or no-JS page extraction as proof that Apple Developer documentation was read. Apple Developer pages often require JavaScript-rendered payloads; a search snippet, empty body, metadata shell, or URL alone is not enough.
- Do not claim a web-doc result was used unless the response includes the actual source link and the relevant documentation text was read through a usable source.
- Keep official web docs as a fallback or explicit preference path rather than the default when Xcode MCP docs are available.
