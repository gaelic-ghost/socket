# v3.1.3 Release Notes

## What changed

- hardened shared-host lifecycle ownership so prune maintenance and retained request-monitor work now shut down under explicit service accounting instead of freestanding task bodies
- made runtime startup truly awaitable and added a first-class `healthcheck` command that proves both HTTP and MCP are healthy instead of only checking process liveness
- tightened persisted runtime-configuration handling, request validation, operator-facing errors, and transport snapshot coverage across HTTP, MCP, and the embedded session surface
- exposed `switchSpeechBackend(to:)`, `reloadModels()`, and `unloadModels()` through the embedded session control surface so app-facing runtimes can drive the same model lifecycle controls as the host HTTP and MCP lanes
- added a first-class `launch-agent promote-live` path, clarified explicit tool-executable handling, and improved LaunchAgent-facing diagnostics for staged live-service refreshes
- aligned repository automation around the `xcrun` toolchain lane, added checked-in SwiftFormat and SwiftLint configuration, and ran the formatting pass across the package
- refreshed the resolved runtime graph to [`SpeakSwiftly` `3.0.5`](https://github.com/gaelic-ghost/SpeakSwiftly/blob/v3.0.5/docs/maintainers/v3-0-5-release-notes.md) and [`TextForSpeech` `0.17.0`](https://github.com/gaelic-ghost/TextForSpeech/blob/v0.17.0/docs/releases/v0.17.0.md)

## Breaking changes

- none intended

## Migration or upgrade notes

- prefer `xcrun swift build` and `xcrun swift test` for the standard package lane in this repository; current repo guidance and CI intentionally center the Xcode-selected toolchain path
- use `xcrun swift run SpeakSwiftlyServerTool healthcheck` after LaunchAgent installs or staged-artifact refreshes when you need one concise probe that covers `GET /healthz`, `GET /runtime/host`, and MCP `initialize`
- use `launch-agent promote-live` when the intent is to rebuild the current checkout, stage `.release-artifacts/current`, refresh the staged executable signature, and repoint the live per-user service in one step

## Known issues carried forward

- `mlx-audio-swift` still emits the existing unhandled-`README.md` warning noise during SwiftPM build and test lanes
- the recurring `freed pointer was not the last allocation` playback warning still needs the focused ownership audit tracked in [`ROADMAP.md`](../../ROADMAP.md)
- the repo still needs the maintainer-facing post-tag live-service verification path tracked in [`docs/maintainers/live-service-reliability-follow-ups.md`](../maintainers/live-service-reliability-follow-ups.md)
