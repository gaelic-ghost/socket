# Bullet Quality Guide

## Objectives

- Use concise, professional, matter-of-fact language.
- Keep each bullet at 1-3 sentences.
- State tool/system, action, and positive result.
- Write in third person without repeating a personal name.

## Preferred Pattern

Use this structure when possible:

1. Tool/language/system and scope of work.
2. What was built, improved, or automated.
3. Positive outcome (speed, quality, reliability, adoption, maintainability).

## Strong Verbs

Implemented, designed, delivered, automated, optimized, containerized, orchestrated, integrated, led, aligned, communicated, reviewed.

## Results Vocabulary

Improved reliability, reduced regressions, accelerated delivery, lowered setup friction, increased confidence, improved maintainability, strengthened release quality.

## Collaboration Signals

For collaborative projects, include some bullets that explicitly mention:

- Teamwork and shared ownership.
- Leadership (planning, unblocking, technical direction).
- Communication (tradeoffs, status updates, alignment).

## Avoid

- Vague claims without action or result.
- Long narrative paragraphs.
- First-person phrasing.
- Repeating the same tool or result wording in every bullet.
