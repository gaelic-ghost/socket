## What changed

- Added `maintain-project-accessibility` as the canonical maintainer skill for repository-local `ACCESSIBILITY.md` standards documents.
- Extended `maintain-project-contributing` so ordinary-project contribution guides now carry an explicit `Accessibility Expectations` contributor contract.
- Added maintainer planning and release-surface documentation for the new accessibility workflow, including the canonical section schema and rollout plan.
- Corrected stale workflow-atlas output-field wording so maintainer docs now match the actual README and CONTRIBUTING maintenance scripts.

## Breaking changes

- None.

## Migration/upgrade notes

- If you want repository-local accessibility standards maintenance from this plugin, install and use `maintain-project-accessibility`.
- Repositories using `maintain-project-contributing` should expect `Development Expectations > Accessibility Expectations` to be part of the canonical contribution-guide structure.

Canonical install commands:

```bash
npx skills add gaelic-ghost/productivity-skills --skill maintain-project-accessibility
npx skills add gaelic-ghost/productivity-skills --all
```
