# v4.3.0

## What changed

- Removed the nested `plugins/productivity-skills/` packaging subtree from the repository.
- Retargeted the repo-local Codex and Claude marketplace catalogs to point directly at root `skills/`.
- Updated public and maintainer docs to describe the simpler direct-to-`skills/` discovery model.
- Removed stale references to the retired `install-plugin-to-socket` workflow from current repo guidance.

## Breaking changes

- The tracked nested plugin packaging copy under `plugins/productivity-skills/` no longer exists in this repository.

## Migration/upgrade notes

- Repo-local marketplace discovery should now point directly at `skills/` through `.agents/plugins/marketplace.json` and `.claude-plugin/marketplace.json`.
- Any local tooling or personal notes that still refer to `plugins/productivity-skills/` should be updated to use root `skills/` instead.

Canonical install commands:

```bash
npx skills add gaelic-ghost/productivity-skills --skill explain-code-slice
npx skills add gaelic-ghost/productivity-skills --all
```
