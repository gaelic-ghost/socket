# v4.2.0

## What changed

- Added `maintain-project-contributing` as the canonical maintainer skill for ordinary-project `CONTRIBUTING.md` files.
- Added the repository's own `CONTRIBUTING.md` so public-facing repo positioning can stay in `README.md` while contributor workflow lives in the canonical contribution guide.
- Clarified repo positioning across public and maintainer docs so `productivity-skills` is described as both the globally installable plugin and the canonical general-purpose baseline layer for workflow families that later specialize into narrower plugins.
- Added `maintain-project-repo` install-profile support to make the toolkit more reusable across generic, Swift package, and Xcode app repositories.

## Breaking changes

- None.

## Migration/upgrade notes

- If you want contribution-guide maintenance from this plugin, install and use `maintain-project-contributing`.
- Repository-specific or stack-specific specializations should continue to live in narrower plugins when stronger assumptions materially improve the workflow; `productivity-skills` remains the general-purpose baseline layer.

Canonical install commands:

```bash
npx skills add gaelic-ghost/productivity-skills --skill maintain-project-contributing
npx skills add gaelic-ghost/productivity-skills --all
```
