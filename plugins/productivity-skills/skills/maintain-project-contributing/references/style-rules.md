# Style Rules

- Keep contributor guidance direct, concrete, and repo-grounded.
- Favor short explanatory paragraphs plus small bullet lists when they improve scanability.
- Treat `CONTRIBUTING.md` as a workflow guide for contributors, not as a contributor roster.
- Keep `Local Setup` operational and practical, with explicit `Runtime Config` and `Runtime Behavior` subsections.
- Keep `Naming Conventions` explicit so contributors can extend the repo without drifting terminology or casing.
