# Style Rules

- Keep AGENTS guidance direct, technical, durable, and repo-grounded.
- Prefer concise paragraphs and short bullet lists over sprawling policy prose.
- Treat source-of-truth rules, command guidance, and workflow boundaries as first-class AGENTS content.
- Use explicit section labels for commands, review expectations, and safety boundaries so maintainers and agents can scan quickly.
- Keep the root AGENTS file compact and practical, aligned with official Codex guidance for repo-local instructions.
- Treat routing guidance as a scan aid: point to the few highest-value files or directories first instead of trying to summarize the entire repo.
