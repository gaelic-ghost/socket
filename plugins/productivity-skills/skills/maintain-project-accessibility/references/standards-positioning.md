# Standards Positioning

This baseline skill should position accessibility standards carefully.

Use language like:

- "targets WCAG 2.2 AA"
- "works toward WCAG 2.2 AA"
- "documents the project's accessibility baseline and verification evidence"

Avoid language like:

- "fully compliant"
- "fully conformant"
- "100% accessible"

unless the repository has grounded evidence that truly supports that stronger claim.

This skill documents standards, architecture, verification, and known gaps for maintainers. It is not a substitute for a public accessibility statement or a formal procurement-facing conformance report.
