# Project Accessibility Maintenance Automation Prompts

Use prompts like these when validating the skill's intended trigger surface:

- "Audit this project's ACCESSIBILITY.md and tell me what's missing."
- "Normalize our accessibility doc to the shared template."
- "Create an ACCESSIBILITY.md for this repo from the baseline workflow."
- "Check whether our accessibility standards doc overclaims compliance."
- "Help me tighten the accessibility architecture and testing sections in this repo."

Prompts that should usually route elsewhere:

- public accessibility statement generation
- VPAT or ACR drafting
- stack-specific accessibility implementation reviews that belong in a narrower plugin
